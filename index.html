Introduction

GitHub is a web-based version-control and collaboration platform for software developers.

It uses Git, the distributed version-control system created by Linus Torvalds.

GitHub provides a web-based graphical interface as well as access control and several collaboration features.

git --version
  • Web-based Git repository hosting service
  • Supports Git commands and workflows
  • Popular among open-source communities
Creating a Repository

Repositories are where project files and revision history are stored.

You can create a repository on GitHub by clicking the “New repository” button.

Name your repository and choose its visibility (public or private).

git init git remote add origin https://github.com/username/repo.git
  • Click "New" on the repositories page
  • Fill out the form
  • Initialize with README if needed
Cloning a Repository

Cloning allows you to create a local copy of a remote repository.

This is useful for contributing to existing projects or accessing the code offline.

git clone https://github.com/username/repo.git
  • Navigate to the repo on GitHub
  • Copy the HTTPS or SSH URL
  • Use the git clone command
Committing Changes

Committing saves your changes to the local repository.

Each commit has a unique ID and an optional message.

git add . git commit -m "Initial commit"

Use meaningful commit messages to track changes effectively.

Pushing to GitHub

Pushing sends your local commits to a remote GitHub repository.

This updates the remote branch with your local changes.

git push origin main

Make sure you are authenticated with GitHub before pushing.
